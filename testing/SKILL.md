# Testing Skill

Write and manage comprehensive tests.

## When to Use

- User asks to "write tests", "add test coverage", or "test this function"
- Running existing test suites
- Debugging test failures
- Setting up CI/CD test pipelines
- Test-driven development (TDD)

## Steps

1. Identify the testing framework used (Jest, pytest, RSpec, etc.)
2. Understand the code structure and dependencies
3. Write tests following the framework conventions
4. Ensure proper mocking/stubbing where needed
5. Run tests and verify coverage

## Example Triggers

- "write unit tests for this module"
- "increase test coverage to 80%"
- "run the integration tests"
