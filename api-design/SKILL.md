# API Design Skill

Design and document REST APIs.

## When to Use

- User asks to design a new API endpoint
- Creating or updating OpenAPI/Swagger specifications
- API versioning decisions needed
- RESTful resource modeling
- Request/response schema design

## Steps

1. Understand the business requirement
2. Define resources and their relationships
3. Design endpoint paths following REST conventions
4. Define request/response schemas
5. Document with OpenAPI spec

## Example Triggers

- "design an endpoint for user authentication"
- "create OpenAPI spec for this API"
- "add versioning strategy to our API"
